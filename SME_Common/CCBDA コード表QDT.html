いただいた XSDファイル (BSM版・LHM版) と Excelシート (BSMシート・LHMシート)、さらにご提示いただいた CCBDAルール（拡張・制限可否一覧） を組み合わせると、次の関係が見えてきます。

1. BSMスキーマの qDT（型定義）

  • BSM版 XSD (SMEConsolidatedSelfInvoice_BSM_09-01.xsd) は、業務セマンティックモデルを反映しており、値の表現形式は主に qDT（Qualified Data Type） によって定義されています。

  • qDT は UN/CEFACT Core Component Technical Specification (CCTS) における標準的なデータ型拡張方式で、たとえば：

    • Amount. Type → decimal（通貨額に対して ISO 4217 通貨コードを補助属性として保持）

    • DateTime. Type → ISO 8601 (format 102: YYYY-MM-DD)

    • Code. Type → string（code_list による制約を付与可能）

  • Excelの BSMシート には、これら qDT に対応する code_list や XML datatype が列として記載されており、たとえば「TaxCategoryCode は code_list=UNCL5305, datatype=string」といった表が並んでいる想定です。

👉 ポイント: BSMレベルでは業務的に必要な「型」と「コードリスト」が明示され、任意か必須か (minOccurs) の制御までは定義されていません。

2. LHMスキーマの XML datatype

  • LHM版 XSD (SMEConsolidatedSelfInvoice_LHM_09-01.xsd) は、論理階層モデルを実装したものです。

  • この段階では qDT の抽象的な表現が、XMLスキーマで使える具体的な xs:datatype に落とし込まれています。

    • qDT:AmountType → xs:decimal + currencyID 属性

    • qDT:DateTimeType → xs:date または xs:dateTime

    • qDT:CodeType → xs:string（＋ code_list に基づいた UsageRule や Schematron で制約）

  • Excelの LHMシート には、BSMで定義された qDT がどのように XML datatype にマッピングされたか、さらに code_list との対応が整理されています。

👉 ポイント: LHMレベルでは XMLインスタンスを生成可能な形に変換されるため、maxOccurs や minOccurs の制約も加わり、繰返し数や必須性の削減（ただし拡張は不可）が明示されます。

3. CCBDAルールとの関係

ご提示の表「6.まとめ：拡張・制限可否一覧」に基づくと：

  • 可とされる操作

    • BBIE（基本業務情報要素）の削除（非使用化）

    • BBIEの必須化（minOccurs増加）

    • BBIEの最大繰返し数の削減（maxOccurs減少）

    • Usage Rule による制約追加（例: Schematronで code_list 値制限）

  • 不可とされる操作

    • BBIEの追加（別ABIEとする必要あり）

    • BBIEの型（qDT）の変更（qDT自体を差替えることは不可）

    • 項目順の変更

    • 定義文の置換

👉 したがって、Excelシートの code_list や XML datatype は、CCBDAが許可する「制約の追加」の範囲内で運用されることになります。
例: xs:string として表されるコード要素に対し、BSMで指定された code_list (UNCL5305 など) を Schematronで制約 → これが Usage Rule 追加に相当し、可。

まとめ

  • BSMスキーマ：業務セマンティックモデル。qDT（型定義）＋ code_list が Excel BSMシートで整理される。

  • LHMスキーマ：論理階層モデル。XML datatype に具体化され、Excel LHMシートに反映される。

  • CCBDAルール：

    • qDT の変更や新しいBBIE追加は不可。

    • ただし、繰返し数削減や必須化、Usage Rule追加（code_list制約）は可。

  • 実務的には、BSMのqDT＋code_list定義をベースに、LHMでXML datatypeに落とし込み、CCBDA準拠の制約ルールで業界や地域特化のカスタマイズを行うという流れになります。
